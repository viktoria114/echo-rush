# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

**Name:** Echo Rush
**Engine:** Godot 4 with GDScript
**Description:** Top-down roguelite shooter with 4 student characters.
The player controls Rael (attacker). Companions Lena (mage), Brom (tank),
and Zari (archer) are autonomous NPCs controlled by AI.

Between waves, Echo appears — a merchant NPC whose dialogue is generated
with the Claude API (`claude-sonnet-4-20250514`). The player types in free
language and Echo responds with skill options.

**Stack:** Godot 4, GDScript, Claude API via HTTPRequest node

## Echo / Claude API Integration

Echo is a merchant NPC that appears between waves. Dialogue is driven by
`claude-sonnet-4-20250514` via Godot's HTTPRequest node (no plugin needed).

**Keyword system:** words like `[FIRE]`, `[BLOOD]`, `[SHIELD]` inside
Echo's response activate real in-game skills. Parse Echo's reply for
bracketed keywords and trigger the corresponding skill effect.

## Architecture

```
/scenes/levels/       — level scenes (.tscn)
/scenes/ui/           — HUD, menus, Echo dialogue UI
/scenes/characters/   — player and companion scenes
/scenes/enemies/      — enemy scenes
/scripts/player/      — Rael input, movement, attack logic
/scripts/enemies/     — enemy AI and behavior
/scripts/ui/          — HUD scripts, Echo dialogue handler
/scripts/systems/     — wave manager, keyword parser, skill system
/assets/sprites/      — sprite sheets and textures
/assets/audio/        — music and SFX
/assets/fonts/        — bitmap/TTF fonts
```

**Godot project conventions:**
- Scenes (`.tscn`) define the node tree; scripts (`.gd`) attach behavior to nodes.
- `res://` is the project root path used within Godot.
- Autoloads (singletons) are registered in `project.godot` under `[autoload]` and act as global managers.
- The `.godot/` directory is generated and gitignored — do not edit it.
- Asset import metadata (`.import` files) are auto-generated by the editor; commit them alongside their source assets.

## Coding Rules

- **Never use C#** — GDScript only.
- Keep scripts short and modular: one responsibility per script.
- Prefer signals over direct node references for decoupled communication.

## Common Commands

```bash
# Run headlessly (export / tests)
godot --headless

# Open in editor
godot -e

# Run a specific scene
godot res://path/to/scene.tscn

# Export release build
godot --export-release "Windows Desktop" build/echo-rush.exe
```
